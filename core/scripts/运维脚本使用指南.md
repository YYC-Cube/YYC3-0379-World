## 📚 详细使用指南

### health-check.sh - 快速健康检查

**用途**: 快速检查各端关键服务状态

**特点**:
- ⚡ 快速执行（< 10秒）
- 🎯 只检查关键服务
- 📊 简洁输出

**使用场景**:
- 日常巡检
- 快速诊断问题
- 自动化监控

**示例**:
```bash
# 快速检查
./health-check.sh

# 定时任务（每5分钟检查一次）
*/5 * * * * /path/to/health-check.sh >> /var/log/yyc3-health.log
```

---

### check-all-nodes.sh - 全面状态检查

**用途**: 深度检查各端系统状态和服务详情

**检查内容**:
- ✅ SSH 配置
- ✅ 节点连通性
- ✅ 数据库状态（PostgreSQL、Redis）
- ✅ Docker 容器
- ✅ 系统资源（CPU、内存、磁盘）
- ✅ 网络配置

**使用场景**:
- 深度诊断
- 问题排查
- 系统审计
- 性能分析

**示例**:
```bash
# 全面检查
./check-all-nodes.sh

# 保存检查报告
./check-all-nodes.sh > /tmp/yyc3-status-$(date +%Y%m%d).txt
```

---

### yyc3-ops.sh - 快速运维工具

**用途**: 一站式运维管理工具

**功能列表**:

| 命令 | 说明 | 示例 |
|------|------|------|
| `status` | 检查节点状态 | `./yyc3-ops.sh status yyc3-33` |
| `services` | 检查服务状态 | `./yyc3-ops.sh services yyc3-22` |
| `docker` | 检查 Docker 容器 | `./yyc3-ops.sh docker all` |
| `logs` | 查看服务日志 | `./yyc3-ops.sh logs yyc3-33 nginx` |
| `restart` | 重启服务 | `./yyc3-ops.sh restart yyc3-33 docker` |
| `update` | 更新系统 | `./yyc3-ops.sh update yyc3-33` |
| `backup` | 备份数据 | `./yyc3-ops.sh backup yyc3-22` |
| `shell` | SSH 连接 | `./yyc3-ops.sh shell yyc3-45` |

**使用示例**:

```bash
# 检查所有节点状态
./yyc3-ops.sh status all

# 检查 ECS 的 Docker 容器
./yyc3-ops.sh docker yyc3-33

# 查看 MacBook 的 PostgreSQL 日志
./yyc3-ops.sh logs yyc3-22 postgresql

# 重启 ECS 的 nginx
./yyc3-ops.sh restart yyc3-33 nginx

# 备份 NAS 数据
./yyc3-ops.sh backup yyc3-45

# SSH 连接到 ECS
./yyc3-ops.sh shell yyc3-33
```

## 🔧 高级用法

### 1. 批量操作

```bash
# 批量检查所有节点的 Docker 状态
for node in yyc3-22 yyc3-33 yyc3-45 yyc3-77; do
    echo "=== $node ==="
    ./yyc3-ops.sh docker $node
done

# 批量更新所有节点
./yyc3-ops.sh update all

# 批量备份所有节点
./yyc3-ops.sh backup all
```

### 2. 自动化监控

```bash
# 创建定时任务
crontab -e

# 添加以下内容
# 每5分钟健康检查
*/5 * * * * /path/to/health-check.sh >> /var/log/yyc3-health.log

# 每天凌晨2点全面检查
0 2 * * * /path/to/check-all-nodes.sh > /var/log/yyc3-status-$(date +\%Y\%m\%d).txt

# 每周日凌晨3点备份
0 3 * * 0 /path/to/yyc3-ops.sh backup all
```

### 3. 告警集成

```bash
# 健康检查并发送告警
./health-check.sh | grep -E "✗|⚠" && \
    curl -X POST "https://your-webhook-url" \
    -H "Content-Type: application/json" \
    -d '{"text":"YYC³ 服务异常，请检查！"}'
```

## 📊 输出解读

### 状态标识

| 标识 | 含义 | 处理建议 |
|------|------|---------|
| ✓ | 服务正常 | 无需处理 |
| ✗ | 服务异常 | 立即处理 |
| ⚠ | 警告状态 | 尽快处理 |

### 常见问题

#### 1. PostgreSQL 未运行

**症状**: `✗ PostgreSQL 未运行`

**处理**:
```bash
# 检查 PostgreSQL 状态
ssh yyc3-22 "systemctl status postgresql"

# 启动 PostgreSQL
ssh yyc3-22 "sudo systemctl start postgresql"

# 查看日志
./yyc3-ops.sh logs yyc3-22 postgresql
```

#### 2. Redis 未运行

**症状**: `✗ Redis 未运行`

**处理**:
```bash
# 检查 Redis 状态
ssh yyc3-22 "systemctl status redis"

# 启动 Redis
ssh yyc3-22 "sudo systemctl start redis"

# 测试连接
ssh yyc3-22 "redis-cli ping"
```

#### 3. API 服务不可访问

**症状**: `✗ API (8000) 未运行`

**处理**:
```bash
# 检查 API 服务
ssh yyc3-33 "docker ps | grep api"

# 重启 API 服务
ssh yyc3-33 "cd /path/to/project && docker-compose restart api"

# 查看日志
./yyc3-ops.sh logs yyc3-33 api
```

#### 4. NFS 挂载失败

**症状**: `✗ NFS 未运行`

**处理**:
```bash
# 检查 NFS 服务
ssh yyc3-45 "systemctl status nfs-server"

# 启动 NFS
ssh yyc3-45 "sudo systemctl start nfs-server"

# 检查挂载点
ssh yyc3-22 "mount | grep nfs"
```

## 🔐 安全建议

### 1. SSH 密钥管理

```bash
# 确保 SSH 密钥权限正确
chmod 700 ~/.ssh
chmod 600 ~/.ssh/id_rsa
chmod 644 ~/.ssh/id_rsa.pub
chmod 644 ~/.ssh/config
```

### 2. 脚本权限

```bash
# 只有所有者可以执行
chmod 700 *.sh

# 或者允许组内用户执行
chmod 750 *.sh
```

### 3. 日志管理

```bash
# 定期清理日志
find /var/log/yyc3-*.log -mtime +30 -delete

# 日志轮转配置
cat > /etc/logrotate.d/yyc3 <<EOF
/var/log/yyc3-*.log {
    daily
    rotate 7
    compress
    missingok
    notifempty
}
EOF
```

## 🆘 故障排除

### 脚本无法执行

```bash
# 检查权限
ls -l *.sh

# 添加执行权限
chmod +x *.sh

# 检查脚本格式
file *.sh
```

### SSH 连接失败

```bash
# 检查 SSH 配置
cat ~/.ssh/config

# 测试连接
ssh yyc3-33 "echo 连接成功"

# 检查密钥
ssh -v yyc3-33
```

### 命令未找到

```bash
# 检查命令路径
which docker
which docker-compose

# 添加到 PATH
export PATH=$PATH:/usr/local/bin
```
